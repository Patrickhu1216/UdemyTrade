
一、 核心目標與背景

- 主題： 使用 Python 構建「資產配對輪動交易策略」。
- 標的： 在 SPY（市值權重標普 500 ETF）與 RSP（等權重標普 500 ETF）之間動態切換。
- 績效： 該策略在過去 20 多年的歷史回測中，展現了顯著的 Alpha（超額回報）。

-核心宗旨： 不只是給一個賺錢的公式，而是掌握「研發流程（Process）」，
          以此為基礎自行測試其他資產配對。

二、 核心交易主體：SPY vs. RSP

策略的根基來自於兩者「指數編製機制」的根本差異：

- SPY (市值加權 - Market-Cap Weighted)：
    - 邏輯： 公司市值越大（股價 X 發行股數），權重越高（如 Apple, Microsoft, Nvidia）。
    - 特性： 本質上偏向「動能（Momentum）策略」，強者恆強。在牛市後期、由少數科技巨頭
    （Mega-caps / FANG）領漲的極端行情中表現極佳。
    - 缺點： 風險高度集中於少數幾檔股票。當市場面臨估值過高的修正（均值回歸）時，跌勢會非常劇烈。
    
- RSP (等權重 - Equal-Weighted)：
    - 邏輯： 500 檔成分股無論大小，每檔初始權重皆相同（約各佔 0.2%），每季定期再平衡（Rebalance）。
    - 特性： 隱含「低買高賣」機制（再平衡時必須賣掉漲多的、買進跌深的）。它更能反映中小型股（Mid/Small-caps）的
      表現。
    - 優勢時機： 當市場呈現普漲行情、或是資金從科技巨頭撤出並輪動到其他板塊（板塊輪動）時，表現通常優於 SPY。
    
- 思維訓練：不要爭論 SPY 或 RSP 哪一個「更好（Better）」。任何投資評估都必須建立在「特定情境（Context）」與
 「時間週期（Time Period）」之上。 量化交易者的任務是找出「什麼時候該持有誰」，而不是尋找單一的完美資產。

三、 量化策略研發 6 大步驟 (Framework)
頂尖對沖基金（Hedge Funds）相同的專業研發工作流：

1. 探索性數據分析 (EDA) ➔ 
2. 定義動態關係 (Z-Score) ➔ 
3. 制定交易規則 ➔ 
4. 加入波動率濾網 ➔ 
5. 歷史回測 ➔ 
6. 績效對比 (對抗基準)

1. 探索性數據分析 (EDA)： 利用圖表、表格與統計數據，觀察 SPY 與 RSP 之間的歷史行為模式。

2. 確定與視覺化動態關係： 利用滾動報酬率（Rolling Returns）與 Z-Score（標準分數）等量化指標，
   捕捉兩者表現的相對強弱與偏離程度。

3. 制定交易規則（Trading Rules）： 明確定義何時將資金完全切換至 RSP，何時切換至 SPY。

4. 引入波動率濾網（Volatility Overlay）： 學習如何利用波動率指標優化策略，在市場極端恐慌時離場，
                                        以降低最大回撤並保護本金。

5. 執行回測（Backtesting）： 在 Google Colab 環境下使用 Python 執行歷史數據模擬。

6. 績效評估（Performance Evaluation）： 將策略成果與全球最通用的基準--「SPY 買入持有策略（Buy and Hold）」
                                       進行勝負對決。
